# Governance

Tythe is a personal project maintained by
[@tamimbinhakim](https://github.com/tamimbinhakim). That's the whole
governance story today. This document is honest about that rather than
pretending there's a foundation behind it.

## Who decides what

I do.

That includes: what gets merged, what versions get cut, what's on the
roadmap, what's out of scope. I try to make those calls in public — in
PR threads, in issues, in [Discussions][disc] — so you can see the
reasoning and push back. When I'm wrong, I want to know about it.

[disc]: https://github.com/tamimbinhakim/tythe/discussions

## How decisions get made

| Kind                                            | How                                                                             |
| ----------------------------------------------- | ------------------------------------------------------------------------------- |
| Typo / docs fix / dep bump                      | I self-merge.                                                                   |
| Bug fix / small feature                         | PR + I review + I merge.                                                        |
| New public API                                  | Open an issue first. Discuss. Then PR.                                          |
| Breaking change                                 | Open an issue. Discuss for at least 7 days unless trivial. Then PR. Bump major. |
| Governance change (this doc, license, transfer) | Announced publicly with at least 14 days for community feedback.                |

## How you can help shape it

The highest-leverage thing you can do is **open an issue that starts
with "I tried to use Tythe for X and got stuck because Y."** That's
worth more than a feature request, more than a PR for a feature I
didn't ask for, more than a vote on a GitHub poll.

Real adoption signal beats abstract preference every time.

## What happens if I disappear

The repo is MIT-licensed. Fork it. That's the escape valve every open
source project has, and it's the point.

If you want to be added as a co-maintainer because you're contributing
substantively over a sustained period, open an issue and we'll talk.

## Conduct

The [Code of Conduct](./CODE_OF_CONDUCT.md) applies to everyone,
including me. If I act like a jerk, call me on it — privately or
publicly.
